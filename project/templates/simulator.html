Password Attack Simulator

A simple attack simulation tool that estimates the time (worst case and average) it would take different attack types to crack the submitted password and displays the results.

About:

In the real world, attackers may use a combination of multiple attack types at the same time to speed up the overall attack time. For instance, they may check the password against a wordlist before commencing a brute force attack.

Brute Force Attack - systematically tries every possible character combination within a defined character set. Simulation returns estimated time based on total search space.

Dictionary Attack - tests the password against a list of commonly used passwords. If a match is found, the password is considered vulnerable and an estimated crack time is returned.

Brute Force x Dictionary Hybrid Attack - also known as a Suffix-Prefix Attack, takes common passwords from a wordlist and systematically adds short sequences (typically 1-3 characters) of numbers or symbols to its beginning and/or end. If a match is found, returns estimated time.

Rule-Based Mutation Attack - applies common transformations (e.g. Leetspeak character substitution, capitalisation, symbol replacement) to dictionary words and checks if any variant matches the password. If a match is found, returns estimated time.

Assumptions:

  • Assumes benchmark speeds for three varying levels of computational hardware power that an attacker may possess: Online Attack Scenario - one thousand guesses per second (dependent on network latency and rate limiting of service), Offline Attack Scenario (with high-end GPU) - ten billion guesses per second, and Specialised Rig Scenario (cracking arrays, botnets, GPU clusters, etc.) - one hundred trillion guesses per second.
  • Utilises a standard top 10,000 most common passwords list to be used in dictionary attack estimate and hybrid attack estimate.
  • Hybrid attack simulation assumes the attacker would consider prefixes and suffixes to have a padding depth of up to 3 characters.
  • Rule-based Mutation attack simulation model assumes every word in the wordlist has an average of 20 rule-based mutations each to determine total candidate pool size.
  • Assumes a charset of 95 ASCII characters for brute force attack estimations.
  • Assumes a total of 42 possible ASCII symbols and digits (not including the space character) are included in the charset used to determine what is and what isn't a special character in the hybrid attack model.
